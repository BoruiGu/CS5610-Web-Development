Post-course - Override

This experiment explores how styles override each other.

#1: Inline style

            
As expected, inline style will override style from a css file.
            

html:
<span id="T1" style="color: black">
As expected, inline style will override style from a css file.
</span>

css:
#T1 {
    color: crimson;
}

#2: Single CSS

        
In a single CSS file, style that comes last "wins".
            

css:
#T2 {
    color: black;
}
#T2 {
    color: white;
}
#T2 {
    color: crimson;
}

#3: Multiple CSS Files

        
When multiple CSS files are included, style in latter file override style in former file.
            

html:
<link rel="stylesheet" href="css/override1.css" />
<link rel="stylesheet" href="css/override2.css" />
css:
override1.css
#T3 {
    color: crimson;
}
override2.css
#T3 {
    color: black;
}

#4: Specificity

            
More specific selectors will override more general ones. (Note: inline styles have high specificity. See the spec below.)

CSS Specification: link
            

html:
<span id="T4">
More specific selectors will override more general ones.
</span>
css:
override1.css
pre #T4 {
    color: crimson;
}
#T4 {
    color: white;
}
override2.css
#T4 {
    color: black;
}

#5: !Important

        
!important at the end of a style will override any other styles, including inline styles.
            

html:
<span id="T5" style="color: crimson">
!important at the end of a style will override any other styles, including inline styles.
</span>
css:
override1.css
#T5 {
    color: black !important;
}
#T5 {
    color: white;
}
override2.css
#T5 {
    color: green;
}
